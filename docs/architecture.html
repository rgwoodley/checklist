Checklist : Architecture

The checklist application utilizes a multi-tier web application architecture, as show in the diagram below. The various tiers/layers serve as incremental building blocks of the application, with each tier only having knowledge of the tier immediately below it.

  • Database
  • Search Index
  • Hibernate
  • Lucene
  • Spring Framework
  • DAOs
  • Services
  • Spring Controllers
  • REST Services
  • JSP
  • JQuery/Backbone.js

  • Database

    Any relational database supported by Hibernate can be used.

  • Search Index

    The application uses Lucene, and can either store Lucene's search index on disk or in memory. The in-memory option is primarily used for testing.

  • Lucene

    Lucene is an open source search engine that provides document/object indexing and ranked index searches.

  • Hibernate

    Hibernate is an ORM (Object-to-Relational Database Mapper) that serves as the interface between the DAO layer and the database.

  • Spring Framework

    The Spring Framework provides a lightweight container in which Spring-managed beans exist and execute. Most application DAOs, Service classes and view controllers are managed by the Spring Framework.

  • DAOs

    The purpose of Data Access Objects (DAOs) is to serve as a layer between persistent data sources and the application's service layer. The DAO layer contains Hibernate and Lucene-specific logic so that higher layers can be implementation-agnostic. Interfaces are defined for DAOs so that implementations can be conveniently changed later (if Hibernate or Lucene were to be replaced, for example).

  • Services

    Service classes contain the core business logic of the application. The features a service provides are consumed by view controllers, and a service accesses persistent data through DAO classes. Interfaces are defined for service classes so that implementations can be conveniently changed later.

  • Spring Controllers

    The application uses the Spring MVC web framework, which is an implementation of the Model-View-Controller web application concept. The model is defined by domain classes and business logic/features provided by service classes. The view is defined using Java Server Pages (JSP) or Javascript that consumes data through REST interfaces. The controllers are the classes in this tier that are responsible for accepting user requests, executing the appropriate business logic, and send the desired output to the view.

  • REST Services

    Some client-side Javascript components make calls to the application via its REST (Representational State Transfer) interface over HTTP. The REST interface takes requests in JSON format and returns the result in JSON. The Backbone.js Javascript framework makes use of the REST interface.

  • JSP

    Java Server Pages (JSPs) make up the majority of the view for the application's user interface. Pages are separated into components using the Apache Tiles templating framework.

  • JQuery/Backbone.js

    Most JSPs contain client-side Javascript, and the application makes heavy use of JQuery. The Backbone.js framework is used on pages where very dynamic behavior is desired.
