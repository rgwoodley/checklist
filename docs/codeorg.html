Checklist : Code Organization

The Checklist codebase conforms to the directory structure of a Maven web application.

  • Root-Level Directories and Files
  • Configuration Files
  • Spring Configuration Files
  • Key Java Packages
  • Key Web Components and Directories
  • Key JSP Directories and Files
  • Key Testing Directories and Files

  • Root-Level Directories and Files

    database
    Database scripts to create and populate database objects.
    docs
    Checklist project documentation in HTML format.
    src
    Source code and related files (HTML, JSP, CSS, Javascript, etc).
    checkstyle_config
    Checkstyle rule configuration file.
    pom.xml
    Maven Project Object Model file.
    suppressions.xml
    Checkstyle rule suppression configuration.

  • Configuration Files
    Located in src/main/resources.

    checklist.properties
    General configuration for the application. Only this file and password.properties need to be modified to get the application up and running.
    log4j.properties
    Logging configuration. Set the logging level here!
    password.properties
    Database password. Set the permissions of this file appropriately!
    quartz.properties
    Properties for the Quartz Job Scheduler.
    tiles-defs.xml
    Tiles configuration for application JSP pages.

  • Spring Configuration Files
    Located in src/main/resources/spring-config.

    checklist-config.xml
    Defines a PropertyPlaceholderConfigurer, which allows values in other Spring configuration files to be replaced by values defined in the checklist.properties and password.properties files.
    checklist-data.xml
    Hibernate configuration.
    checklist-jdbc.xml
    Database/datasource configuration.
    checklist-props.xml
    Configures the 'applicationProperties' bean, which provides properties that can be used in project's Java code.
    checklist-security.xml
    Spring security configuration.
    checklist-service.xml
    Spring configuration for dao, service-tier and asynchronous job classes. NOTE: Almost all Spring service/job behavior can be configured using annotations, so modifications to this file should be rare.

  • Key Java Packages

    com.codeapes.checklist.dao
    "Data Access Objects." The classes in this package and subpackages are responsible for communication from/to a persistent data store.
    com.codeapes.checklist.domain
    This package and subpackages contain classes that define the domain model for the application. You'll find classes like 'User', 'Checklist' and 'UserGroup' in here.
    com.codeapes.checklist.job
    This package and subpackages contain classes that define asynchronous scheduled jobs that are run according to some schedule. An example is the process that refreshes the Lucene search index, making newly indexed objects searchable.
    com.codeapes.checklist.service
    The classes in this package and subpackages define the business logic of the application. The presentation tier uses the methods/features provided by these classes.
    com.codeapes.checklist.web
    This package and subpackages contain Spring MVC controller classes, form classes, and REST services.

  • Key Web Components and Directories
    Located in src/main/webapp.

    css
    Cascading Style Sheets used by the web UI. The application uses CSS 3.
    img
    Graphic images used on the web UI in jpeg, gif or png format.
    index.jsp
    The default/welcome page for the application.
    js
    Application javascript files. The application's client-side javascript makes use of jquery and backbone.js where beneficial.
    WEB-INF/jsp
    Java Server Page (JSP) files live here.
    WEB-INF/checklist-servlet.xml
    Spring MVC configuration for the application. NOTE: Almost all Spring MVC behavior can be configured using annotations, so modifications to this file should be rare.
    WEB-INF/web.xml
    The web application deployment descriptor for the application. This file defines the Spring MVC Dispatcher servlet as well as all required servlet filters, error pages, etc...

  • Key JSP Directories and Files
    Located in src/main/webapp/WEB-INF/jsp.

    includes
    The includes.jsp file in this directory defines commonly used JSP tag libraries. This file should be included in other JSPs if tags from these libraries are needed.
    menu
    Contains the menus that are common throughout the application.
    template
    Commonly-used page components, assembled together using Tiles.
    template/page_template_full.jsp
    The primary Tiles-based template from which most application pages are defined.
    user
    Contains the login page, as well as pages related to user and user profile management.

  • Key Testing Directories and Files
    Located in src/test.

    java
    JUnit tests and supporting Java classes.
    resources/checklist_test.properties
    Testing configuration used in place of checklist.properties when running tests.
    resources/spring-config/checklist-config-test.xml
    Used instead of checklist-config.xml while running tests so that checklist_test.properties is used instead of checklist.properties.
    resources/spring-config/checklist-webutil-test.xml
    Enables Spring annotations to be defined in some web-tier classes without having to load the entire Spring MVC context with a servlet engine running.
    java/com.codeapes.checklist.test.util.TestConfiguration
    This class defines and loads the Spring context so that the test configuration gets used.
