Checklist : Configuration

This page describes the configuration options available in the checklist.properties file. Additional configuration can be changed in the log4j.properties and quartz.properties files. See the project documentation for log4j and the quartz job scheduler for more information about how those components are configured.

  • general

    general.testMode
    Value can be true or false. Enables features that should never be running in a production environment.

  • database

    database.driver
    The JDBC driver name used by the application.
    database.url
    The JDBC URL the application will use to connect to its database.
    database.user
    The username of the user used to connect to the database.
    database.maxIdleConnections
    The maximum number of database connections that can be idle and ready to accept user requests.
    database.maxActiveConnections
    The maximum number of database connections that can be concurrently processing user requests.
    database.validationQuery
    A query to run on a connection to ensure it is still connected to the database. If it is not connected, it will be automatically refreshed.

  • hibernate

    hibernate.showSQL
    Value can be true or false. If set to true, all SQL executed by Hibernate will be sent to the log.
    hibernate.dialect
    The database dialect to use depending on the underlying database, for example MySQLDialect.
    hibernate.hbm2ddl
    Value can be validate, update, create, or create-drop.
    • validate: validate the schema, makes no changes to the database.
    • update: update the database schema if there are hibernate mapping changes.
    • create: create the database schema, destroying previous data.
    • create-drop: same as create, but drops the schema when the application is stopped.

  • search

    search.index.inmemory
    Value can be true or false. If true, the search index is kept in memory-- suitable for testing. If false, the search.index.directory.location property must be set.
    search.index.directory.location
    The path where the search index will be stored. If this directory does not already exist, it will be created.

  • startup

    startup.scheduledjobs.enabled
    Value can be true or false. If true, any jobs that run on a schedule will be enabled with the application starts.

  • auditlog

    auditlog.enabled
    Value can be true or false. If true, all Hibernate write operations will be logged to the audit log.
